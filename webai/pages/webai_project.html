The webAI script records a user’s interactions with the page they are navigating into: mouse movements, touches, clicks and scrolls. Background

In 2021 was presented an initial version of a project titled WebAI. The goal was to analyze whether profiles consisting of gender, emotions or age could be predicted from mouse data movements. A test site was created based on research from Yamauchi & Xiao which created sub experiments in order to provoke emotional or test contextual behaviors. The design of the site was fully thought to trigger those behaviors, but comes with the limitation that most sites are not designed that way and hindered its potential for generalization to other environments.

Additionally, recent improvements in large language models such as ‘Understanding HTML with Large Language Models’ by Gur et all offer possibilities to train such agents in understanding the content it is faced with. In 2016, OpenAI released Universe with the purpose of developing a general purpose agent to play video games and use interfaces. One of its limiting factors was to use video data, which is not the best choice regarding web content. In the scope of this project, we are aiming at using HTML content directly as combined with stylesheets and scripts, contains all the information of the environment. We aim at combining both user’s interaction data on real-world websites with a language model fine-tuned for the web to create autonomous agents.

Several questions are attempted to be answered in the scope of this project

1. What are the most effective machine learning techniques for analyzing mouse data to navigate the web? As most of the data will be time series data from the cursor, deep learning is a favored choice. But possibly algorithms for tabular data such as XGBoost can be used by extracting key metrics from the data, or use several sub-algorithms and voting mechanisms according to what type of action is being done like scrolling, reading, and feed that into smaller SVM models. Training and comparing several such models and highlighting the results will allow us to rank their effectiveness.

2. How well can an AI trained on mouse data collected from real websites navigate and interact with different types of websites? If an AI can be trained to mimic human behavior and complete some tasks on collected web pages, we need to test whether it can do the same on new unseen pages.

3. How can we evaluate the performance of an AI trained on web navigation data? Several performance metrics need to be defined in order to evaluate such an AI, such as the precision of the navigation, identifying elements on the page, or mimicking human behaviors.

4. What are the ethical implications of using mouse data collected from real websites to train an AI for web navigation? Such technology raises questions about online identity, as in this case AIs passing as humans and mimicking their behaviors and interactions can be problematic. These aspects will be arguments towards ethical considerations in this work.

5. How can mouse data collected from real websites be used to predict user profiles? direct navigation data from a real in-situ environment (as not being aware of the experiment), questionnaire to collect personal information about a person, and analysis of their exploration allow to find key metrics linked to certain behaviors and profiles.

More than training Autonomous Agents.

The resulting dataset from the collection would allow to set up a framework for additional works in users’ behavior such as extracting key properties in the data, and eventually propose new routines in the design of landing pages.

How it works

The current phase of this project is to collect data. For that we need to extract interactions users are performing on the web, and you can help by adding the script above on your site. It works this way:

1. Users explore your site normally, each second their mouse movements, clicks and scrolls are sent as packets to a secured server.

2. When collected, the origin page is crawled to extract key metrics from the interaction with the content: specific sections, shapes, colors, all contextualizing the actions performed.

3. After three pages of your site visited, a pop-up invites your visitor to take part in a survey. This allows to map certain profiles with the extracted metrics, and label part of the data.

This process allows linking usage of web-content to a psychological state experienced by the user. Also, we can study these interactions at content-specific levels which could provide insights into improving KPI performances and better design routines for developers.
